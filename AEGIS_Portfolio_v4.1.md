# 🛡️ AEGIS Protocol
## Quantified Connection Firewall — 정량 기반 연결 격리 시스템
**포트폴리오 문서 v4.1**

> 기획 및 설계: Raven
> 구조적 검토 및 수정: Claude (Anthropic)
> 문서 상태: 개념 설계 완료 / 구현 전 단계

---

## 1. 개요 (Executive Summary)

AEGIS는 사이버불링을 '표현의 자유'가 아닌 **디지털 기반 생존권 침해**로 규정하고, 이를 기술적으로 차단하는 연결 격리 시스템이다.

기존 플랫폼 신고 시스템의 한계는 명확하다. 사후 처리, 플랫폼 간 단절, 피해자의 적극적 행동 요구. AEGIS는 이 세 가지를 동시에 해결하는 방향으로 설계됐다.

### 핵심 철학 — 세 줄 요약

```
우리는 사람을 판단하지 않는다.
우리는 콘텐츠를 검열하지 않는다.
우리는 특정 연결(Edge)이 만들어내는 결과만을 차단한다.
```

### 정책 정합성 (Policy Alignment)

2026년 2월 유럽연합(EU)이 발표한 Action Plan Against Cyberbullying은 DSA 미성년자 보호 가이드라인 검토, Trusted Flaggers 역할 강화, 플랫폼의 harmful content 노출 방지 조치를 강조하고 있다. AEGIS는 이러한 정책 방향과 정확히 부합하며, Edge 기반 확률적 격리라는 기술적 접근으로 플랫폼이 systemic risk를 실시간·비례적으로 완화할 수 있는 구체적인 메커니즘을 제공한다.

### 설계의 핵심 전환

| 기존 시스템 | AEGIS |
|---|---|
| 사람 단위 제재 | 관계(Edge) 단위 제한 |
| 사후 처리 | 실시간 격리 |
| 플랫폼 개별 대응 | 크로스 플랫폼 신호 공유 |
| 피해자가 신고해야 작동 | 피해자 노출 전 차단 |
| 이진 판정 (유죄/무죄) | 확률적 리스크 스코어 |
| 영구 제재 | 시간 감쇠 자동 복구 |

## 2. 시스템 정체성

AEGIS는 심판관이 아니다. 처벌 집행자가 아니다.

**AEGIS는 방패다.**

판단은 확률적으로, 개입은 비례적으로, 권한은 인간 감독에 의해 제한된다. 시스템은 유죄를 추정하지 않는다. 리스크를 측정할 뿐이다.

---

## 3. 데이터 모델 — Edge 기반 설계

### 3.1 핵심 개념

모든 상호작용을 **관계 단위**로 모델링한다.

```
Edge(A → B)
A: 발신자
B: 수신자
```

계정 전체를 제재하지 않는다. A→B 관계만을 제한한다. 같은 A가 C, D와 나누는 정상적인 관계는 영향받지 않는다.

### 3.2 Edge Risk Score 모델

```
Risk(A→B) = w1·F + w2·I + w3·P + w4·R
```

| 변수 | 의미 | 초기 가중치 | 설정 근거 |
|---|---|---|---|
| F (Frequency) | 단위 시간 내 메시지 빈도 | 0.25 | 기본 신호 |
| I (Intensity) | 단시간 집중 전송 밀집도 | 0.20 | 순간 폭발성 보조 지표 |
| P (Persistence) | 무응답 상태에서 반복 시도 | 0.35 | 실제 괴롭힘의 핵심 패턴 |
| R (Recurrence) | 차단/무시 이후 재접근 | 0.20 | 재시도 보정 |

**P를 가장 높게 설정한 근거**: 사이버불링 행동 심리 연구에서 일관되게 확인되는 핵심 패턴은 빈도가 아닌 지속성이다. 상대가 반응하지 않아도 멈추지 않는 행동이 실제 피해를 만든다.

### 3.3 가중치 학습 방법

초기값은 행동 심리 기반 휴리스틱으로 설정하되, 실제 데이터로 지속 학습한다.

```python
# 데이터 구조
X = [F, I, P, R]          # 각 Edge의 행동 벡터
Y = {0, 1}                 # 0: 정상, 1: 실제 피해 발생

# 모델 선택: Logistic Regression
# 이유: 설명 가능성 — 오탐 발생 시 역추적 가능
P(risk) = sigmoid(w1·F + w2·I + w3·P + w4·R)
```

**라벨 기준**: 신고 여부가 아닌 **실제 피해 발생 여부**를 Y값으로 사용한다. 신고 기반 라벨은 "신고를 많이 유발하는 행동"을 학습하지만, 피해 기반 라벨은 "실제 피해를 만드는 행동 패턴"을 학습한다. 기존 사이버불링 임상 연구 데이터, 법적 처리 케이스, 피해자 증언 데이터를 골드 스탠다드 라벨로 활용한다.

**클래스 불균형 보정**: 실제 피해 사례는 전체 상호작용의 소수다. SMOTE 오버샘플링 또는 Class weight 조정으로 보정한다.

### 3.3-1 프라이버시 보호 학습 전략

- Edge 데이터는 해시 기반 또는 **federated learning** 방식으로 처리하여 원문 콘텐츠를 중앙 서버에 저장하지 않음
- **On-device Edge AI** (경량 NLP 모델)를 활용한 로컬 Risk 계산 우선 고려 — 메타버스/XR 환경에서 지연 최소화 및 프라이버시 강화에 특히 유효
- 크로스 플랫폼 신호 공유는 익명화된 Risk Score 및 Edge 해시만 교환 — 원문 콘텐츠 및 신원 정보 비공유

이 전략은 "OS 레벨 모니터링 금지"라는 기존 제약과 일관되며, 2026년 Edge AI 트렌드와 연결된다.

### 3.4 시간 감쇠 모델

```
Risk(t) = Risk₀ · e^(-λt)
```

모든 Risk는 시간에 따라 자동 감소한다. 영구 상태는 존재하지 않는다.

- 예: λ = 0.36 설정 시 → 1시간 무활동 → 약 30% 감소
- Risk < 0.4 도달 시 → NORMAL 자동 복귀
- 복귀 후 공격 재개 시 → Risk 재상승 → Stage 재진입

## 4. 상태 머신 (State Machine)

### 4.1 임계값 기반 상태 전이

```
Risk < 0.4   →  NORMAL
0.4 ≤ Risk < 0.7  →  STAGE 1: Silent Filter
Risk ≥ 0.7   →  STAGE 2: Dopamine Reduction
시간 경과 + Risk 감소  →  RECOVERY → NORMAL
```

### 4.2 단계별 동작

**NORMAL**
정상 상호작용. 개입 없음. Risk 측정만 지속.

---

**STAGE 1: Silent Filter** (Risk 0.4–0.69, 자동 실행)

- Edge(A→B) 메시지 전달 차단
- A는 정상 전송으로 인지 (쉐도우 필터)
- 전송 딜레이: 60–300초 랜덤 적용
- 읽음 확인 및 참여 신호 비활성화

**목적**: 피해 전달 완전 차단 + 공격 확인 비용 증가

*DSA 미성년자 보호 가이드라인에서 권고하는 'block/mute any user' 및 'consent-based group addition' 원칙과 기술적으로 호환 가능.*

---

**STAGE 2-A: Signal Reduction** (Risk 0.7+, 자동 실행)

- Stage 1 모든 조치 지속
- 응답 신호 최소화
- 노출 가능성 점진적 축소

**목적**: 공격 강화 루프 차단

---

**STAGE 2-B: Feedback Deception** (Risk 0.7+ AND Attachment Score 임계값 초과 AND 72h 지속, 자동 실행)

- 전송 딜레이 극대화
- 읽음 확인 신호 유지 (실제 전달 없음)
- 새로운 콘텐츠 생성 없음 — 기존 신호의 지연 및 희석만 허용

**적용 조건**: 단순 반복 공격자는 Stage 2-A로 충분하다. Stage 2-B는 시스템을 학습하고 우회를 시도하는 적응적 공격자에게만 적용된다. 이들은 공격의 성공 여부를 확인하려는 동기를 가지므로, 확인 비용 극대화가 효과적이다.

**목적**: 지능적 공격자를 격리된 루프에 가두어 에너지 소모 유도

*메타버스/VR 환경에서 특히 효과적 — 공격자가 아바타 간 상호작용 피드백을 확인하지 못하게 하여 몰입형 harassment 루프를 차단한다.*

---

**RECOVERY**

- Risk가 시간 감쇠로 0.4 미만 도달 시 자동 복귀
- 복귀 사실은 로그로 기록
- 재공격 시 즉시 Risk 재계산

## 5. 관계 집착 모델 (가스라이팅 대응)

### 5.1 Attachment Score

```
Attachment(A→B) = a1·T + a2·C + a3·P
T: 대화 지속 시간
C: 반복 접촉 횟수
P: 응답 유도 시도 횟수
```

### 5.2 위험 조건 및 대응

```
Attachment ↑ + Persistence ↑ → 비정상적 관계 지속 감지
→ 해당 Edge에만 Stage 적용
→ 다른 관계에는 무영향
```

---

## 6. 집단 선동 탐지 (Network Layer)

### 6.1 탐지 모델

```
IncomingEdges(B) 분석
탐지 조건:
  - 10분 이내
  - 신규 Edge 5개 이상
  - 동일 대상 B 집중
→ Target Convergence 발생
```

### 6.2 대응 원칙

- 신규 Edge들에만 Stage 1 자동 적용
- 기존 정상 관계는 유지
- 개인 계정 제재 없음 — 흐름만 차단
- Target Convergence 발생 시, 플랫폼 내부 Silent Filter와 함께 **Trusted Flaggers**에게 익명화된 신호를 전달하여 인간 감독을 보완 (DSA Action Plan Trusted Flaggers 가이드라인 Q2 2026 예정과 연계)

**설계 의도**: 조직적 공격에서 개인 표적화 없이 유입 흐름 자체를 차단한다. 표현의 자유 논쟁을 우회하면서 피해를 막는 구조다.

## 7. 검증 및 성능 지표

### 7.1 임계값 설정 방법

가중치만큼 임계값도 데이터로 결정한다.

```
ROC Curve 기반 트레이드오프 설정
목표:
  FP (정상인데 차단) → 낮게 유지 — 사용자 경험 보호
  FN (가해인데 통과) → 적당히 허용 — 완벽한 탐지보다 안정적 운영
```

### 7.2 핵심 성능 지표 (KPI)

정확도(Accuracy)는 이 시스템의 평가 지표가 아니다. 전체 상호작용 중 사이버불링이 소수인 구조에서 정확도는 의미 없는 숫자가 된다.

```
Recall (재현율): 실제 사이버불링 중 탐지한 비율
Precision (정밀도): 탐지한 것 중 실제 사이버불링 비율
F1 Score: Recall과 Precision의 조화 평균
False Positive Rate: 정상 사용자 차단 비율
```

### 7.3 A/B 테스트 구조

```
Control 그룹: 기존 플랫폼 시스템
Test 그룹: AEGIS 적용

측정 지표:
  Safety: 신고율 감소, 차단 발생률 감소
  Engagement: 정상 유저 체류 시간, 메시지 지속률
  Precision: False Positive Rate

판단 기준: "유해 감소 + 정상 유지" 동시 만족
```

### 7.4 온라인 러닝 (지속적 학습)

```
피드백 데이터:
  - 유저 차단 행동
  - 신고
  - 대화 종료율
  - 이탈률
  - 최신 피해 사례 (지속 수집)

업데이트 방식:
  - 주기적 재학습 (주 1회)
  - Drift 감지 시 즉시 재조정
```

## 8. 시스템 제약 조건

### 허용

- 관계 단위 제한
- 자동 복구 (시간 감쇠)
- 플랫폼 간 해시 기반 신호 공유
- 피해자 측 앱 레이어 필터링
- 프라이버시 우선 학습 (federated learning, on-device inference, hash-only signal sharing)

### 금지

- 계정 전체 제재
- 영구 제한 (법원 명령 없이)
- OS 레벨 / 커널 레벨 디바이스 모니터링
- 의미 / 의도 기반 콘텐츠 판정
- 이진 유죄 판정 출력
- 피해자 동의 없는 개입

## 9. 설계 한계의 명시적 인정

AEGIS는 완성된 시스템이 아니다. 이것은 설계 실패가 아니라 **현실 인식의 반영이다.**

### 현재 구현을 막는 선행 조건

| 조건 | 현황 (v4.1) |
|---|---|
| AI 선제 개입을 허용하는 법적 프레임워크 | EU DSA Action Plan(2026.2) 및 미성년자 보호 가이드라인 검토 진행 중 — 2026년 내 구체화 예상 |
| 플랫폼 간 API 협력 의무화 | DSA systemic risk mitigation 및 Trusted Flaggers 가이드라인 통해 부분 진전 |
| AI 개입 시스템에 대한 사회적 합의 | 형성 중 — 피해자 중심 최소 개입 설계로 논란 최소화 |

### 현실적 출발점

현재 기술과 정책 환경에서 AEGIS는 Stage 1-2 자동화 구간만 즉시 구현 가능하다. 과거 피해 패턴 학습 기반으로 현재 사이버불링의 상당 비율을 차단하는 것이 1차 목표다. 완벽한 시스템은 목표가 아니다. 현재보다 나은 시스템이 목표다.

특히 미성년자 보호를 위한 DSA 가이드라인 검토(2026 예정)와 연계하여 Stage 1 Silent Filter부터 우선 구현하는 것이 현실적인 로드맵이다.

## 10. 설계 진화 과정 (포트폴리오 서사)

이 문서의 가치는 최종 결과물만이 아니다. v1에서 v4.0까지의 설계 진화가 함께 제출된다.

| 버전 | 핵심 변화 | 변화의 이유 |
|---|---|---|
| v1.0 | 무결한 AI 심판관, 영구 디지털 추방 | 초기 기획 |
| v2.0 | 확률적 리스크 스코어, 인간 게이트 도입 | 오류 가능성 내재화 |
| v3.0 | 피드백 기만 재분류 (금지→조건부) | 처벌이 아닌 억지력으로 재규정 |
| v4.0 | Edge 기반 모델, ML 학습 구조, Stage 분리 | 구현 가능성 중심 재설계 |
| v4.1 | Triple Safety Layer, EU DSA 정합성, federated learning, XR 적합성 | 정책·프라이버시·확장성 보강 |

**각 버전 전환의 공통점**: 비판을 수용하되, 설계의 핵심 동기 — 피해자 보호 — 는 한 번도 바뀌지 않았다.

## 11. 설계 철학 (Design Philosophy)

> 사이버불링은 표현의 자유의 문제가 아니다. 생존권의 문제다.
>
> 이 시스템의 목표는 인터넷을 예의바르게 만드는 것이 아니다.
> 어떤 사람도, 자신을 파괴하려는 자들이 가진 도구보다
> 느리고 약하고 파편화된 도구 때문에 무너지지 않도록 하는 것이다.
>
> AEGIS는 완벽한 세계를 만들려 하지 않는다.
